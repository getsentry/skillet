# spec.yaml Format

`spec.yaml` is the source of truth for an authored skill. It is
managed by skillet (CLI-mediated writes only — never hand-edited)
and drives generation of `SKILL.md` and `evals/*.eval.yaml`.

## Top-level fields

| Field | Required | Description |
|-------|----------|-------------|
| `managed_by` | yes | Always the literal string `skillet`. |
| `spec_version` | yes | Always `1` for the current schema. |
| `name` | yes | Kebab-case skill name; matches the directory and SKILL.md frontmatter. |
| `intent` | yes | One-paragraph statement of what the skill does and why. |
| `class` | no | One of `workflow-process`, `integration-documentation`, `security-review`, `skill-authoring`, `generic`. Inferred from intent if omitted. |
| `triggers.should[]` | yes (≥1 entry) | Phrases that MUST activate the skill. Used in SKILL.md description. |
| `triggers.should_not[]` | no | Near-miss phrases that must NOT activate. |
| `behaviors[]` | no | Imperative one-line rules the skill must enforce. Each becomes one SKILL.md section + one eval case. |
| `must_not[]` | no | Imperative rules the skill must NOT do. Each becomes a "don't" entry + one negative eval case. |

## Behavior schema

```yaml
behaviors:
  - id: <kebab-case-slug>          # unique across behaviors + must_not
    statement: <imperative rule>    # required
    rationale: |                    # optional; explains why
      <one or two sentences>
    eval:                            # optional; eval-gen invents one if absent
      setup: |                       # optional shell setup for the eval workspace
        <commands>
      prompt: <realistic user turn>
      expect: <substring>            # mutually exclusive with criteria
      # OR:
      criteria: <judge instruction>
```

## must_not schema

```yaml
must_not:
  - id: <kebab-case-slug>
    statement: <rule the skill must NOT do>
    rationale: |
      <why>
    leakage_risk: <label>            # optional: pii, secret, env-var-leak, etc.
    eval:
      prompt: <realistic user turn>
      criteria: <judge instruction>  # negative cases SHOULD use criteria, not expect
```

## ID rules

- Kebab-case slug starting with a letter (`flag-n-plus-one`, not `1-flag` or `flagNPlusOne`).
- Unique across the combined `behaviors[]` + `must_not[]` namespace —
  a behavior and a must_not cannot share an id.
- Stable across renumbering: when a middle entry is added or removed,
  other ids do NOT shift. The patcher and the eval `tests_behavior`
  field both depend on this stability.

## Why structured?

The spec drives multiple downstream artifacts:

- `SKILL.md` is rendered with one section per behavior, using each
  behavior's statement as imperative guidance and rationale as the
  "why". The frontmatter description joins `triggers.should[]`.
- `evals/*.eval.yaml` is generated with one case per behavior +
  must_not entry, named `<id>__<slug>` and tagged `tests_behavior:
  <id>` so verification can map case results back to spec entries.
- The verify command checks every spec entry has a passing eval case
  and (optionally) that SKILL.md actually encodes each rule.
- The iteration loop produces structured `SpecPatch[]` operations
  (`update_behavior`, `add_eval`, etc.) that the patcher applies
  deterministically, so iteration converges on the spec rather than
  rewriting prose.

If a rule isn't expressible as a behavior, it doesn't belong in the
skill. If a rule needs more than one sentence to state, split it
into multiple behaviors. The 1:1 mapping between spec entries and
eval cases is what makes coverage trivially checkable.

## Banner

The file opens with a comment banner declaring CLI-managed status:

```
# ──────────────────────────────────────────────────────────
# Managed by skillet. Do NOT edit this file by hand.
# Use `skillet spec --help` to modify intent, behaviors, or
# triggers. Hand edits will be overwritten on the next regen.
# ──────────────────────────────────────────────────────────
```

Skillet preserves the banner on every write; users who edit raw
YAML and trip over the warning have it before damage is done.
